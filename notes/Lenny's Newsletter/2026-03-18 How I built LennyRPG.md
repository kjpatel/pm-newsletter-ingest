# How I built LennyRPG

**Source**: [Lenny's Newsletter](https://www.lennysnewsletter.com/p/how-i-built-lennyrpg-f85)
**Author**: Lenny Rachitsky | **Date**: Mar 18, 2026

## Summary

Ben Shih, a non-technical designer at Miro, shares his six-step workflow for building LennyRPG, a Pokémon-style game that lets users battle podcast guests with product trivia, demonstrating how AI tools like Claude Code can enable designers to ship complex projects without coding experience.

## Key Takeaways

- **Use AI interviews instead of PRDs** – Let Claude Code interview you about your project vision rather than writing traditional product requirement documents, which can be faster and more conversational.
- **Know when to pivot your technical framework** – Be willing to switch tools if your initial choice doesn't fit the problem, rather than forcing the wrong solution to work.
- **Systematize data processing at scale** – Develop repeatable workflows to efficiently convert large datasets (300+ transcripts) into usable game data instead of manual approaches.
- **Leverage no-code and low-code tools strategically** – Non-technical makers can ship complex interactive products by combining Claude Code, Phaser 3, Supabase, and other accessible platforms.
