# “I haven’t written a single line of front-end code in 3 months”: How Notion’s design team uses Claude Code to prototype

**Source**: [Lenny's Newsletter](https://www.lennysnewsletter.com/p/i-havent-written-a-single-line-of)
**Author**: Lenny Rachitsky (featuring Brian Lovin) | **Date**: Feb 23, 2026

---

## Summary

Brian Lovin from Notion demonstrates how Claude Code has transformed the design team's prototyping process by enabling designers to build functional prototypes collaboratively without writing traditional front-end code. By creating a shared Next.js environment and leveraging AI-assisted coding, Notion's design team has dramatically accelerated iteration and discovered that code-based prototyping is essential for designing AI-powered products effectively.

## Key Takeaways

- **Build a shared prototype playground** using Next.js and Claude Code where your entire design team can collaboratively create, iterate, and deploy functional prototypes without siloed repositories or static design tools.
- **Encounter reality early** by prototyping with working code rather than static mockups, which surfaces constraints and possibilities that static design tools like Figma cannot reveal.
- **Teach Claude to automate repetitive tasks** by creating custom slash commands and skills—when Claude suggests a workflow improvement, formalize it as a reusable skill rather than repeating manual steps.
- **Use Claude Code's plan mode** to get better results when prototyping, and leverage prompting techniques to guide AI when it's not following your intentions.
- **Transform Figma designs into production code** with single prompts, then iterate on working prototypes rather than treating design and development as sequential phases.
