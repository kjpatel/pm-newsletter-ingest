# 🎙️ This week on How I AI: How Notion’s design team uses Claude Code to design

**Source**: [Lenny's Newsletter](https://www.lennysnewsletter.com/p/this-week-on-how-i-ai-how-notions)
**Author**: Lenny Rachitsky (featuring Brian Lovin) | **Date**: Feb 23, 2026

---

## Summary

Notion's design team uses Claude Code to transform Figma designs into working prototypes in a shared Next.js environment, eliminating the need for designers to write front-end code and enabling them to test designs against real AI models and catch edge cases early.

## Key Takeaways

- **Shift to code-first prototyping**: Move designs from static Figma mockups to interactive code prototypes connected to real AI models to validate feasibility and discover edge cases early.
- **Build shared prototype playgrounds**: Create centralized Next.js environments where entire design teams can discover, reuse, and iterate on prototypes together rather than working in isolated repositories.
- **Teach Claude to automate manual tasks**: When Claude asks you to perform repetitive work, create custom Claude Skills or slash commands to teach it to handle those tasks autonomously instead of doing them manually.
- **Design AI experiences in code, not mockups**: Static design tools cannot capture the actual feel and behavior of AI interactions—test with real models in browsers to understand failure modes, loading states, and user experience.
- **Use custom slash commands for accessibility**: Create commands like '/figma' that encapsulate complex multi-step workflows, making advanced AI techniques usable by less-technical team members.
