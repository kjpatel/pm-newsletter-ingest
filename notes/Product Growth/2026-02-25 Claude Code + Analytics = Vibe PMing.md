# Claude Code + Analytics = Vibe PMing

**Source**: [Product Growth](https://www.news.aakashg.com/p/frank-lee-podcast)
**Author**: Aakash Gupta | **Date**: Feb 25, 2026

---

## Summary

Vibe PMing leverages Claude Code with MCP (Model Context Protocol) to connect AI agents directly to product analytics, feedback tools, and ticketing systems, enabling PMs to automate data analysis, reporting, and spec writing. The article provides a practical guide to setting up these systems and five key workflows that can replace hours of manual PM work.

## Key Takeaways

- **Build a product repo structure** in Claude Code with all PRDs, roadmaps, and specs in Markdown files, then reference them with @ commands to eliminate manual copy-pasting and searching.
- **Create reusable skills** as Markdown files with clear names and heuristics (like /analyze-chart or /analyze-feedback) that enable consistent, repeatable agent workflows rather than one-off experiments.
- **Connect MCPs strategically** to your analytics platform and ticket system with well-configured tool names and descriptions, being selective about which MCPs are active to avoid processing noise and slowing responses.
- **Automate dashboard reporting** by pointing agents at your key dashboards to generate weekly insights and urgent alerts automatically, eliminating the Sunday night ritual of manual metric compilation.
- **Move from insight to spec directly** by piping chart analysis and feedback summaries into Claude Code alongside product screenshots to brainstorm ideas and draft PRDs, rather than starting from a blank document.
