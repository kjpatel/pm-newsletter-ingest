# A Guide to Claude Code for PMs: From Cowork to Code

## Summary

Claude Code extends Claude Cowork's capabilities by adding IDE features and direct codebase access, enabling PMs to prototype working software and manage git workflows without needing deep engineering knowledge. The guide walks through setup, key differences, and practical PM use cases that bridge the gap between planning and building.

## Key Takeaways

- **Install VS Code and Claude Code extension** to access IDE features like file explorer, terminal commands, and git integration that Cowork doesn't provide—no engineering degree required.
- **Leverage Auto Memory** to maintain context across sessions automatically via CLAUDE.md and /memory commands, eliminating the need to re-explain projects daily.
- **Use Plan Mode (Shift+Tab)** before making changes to safely explore codebases and get Claude's approval on proposed approaches before execution.
- **Prototype features instead of writing tickets** by building working demos in Claude Code and pushing to git branches with PRs, replacing traditional Jira specification handoffs.
- **Monitor context usage** with /context visualization to prevent quality degradation—compact conversations at 50% fill to maintain optimal performance.
